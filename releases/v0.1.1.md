# mcp-registry-diff v0.1.1

## What changed

- Fixed keyed registry parsing when server records are mixed with metadata fields.
- Preserved list-valued `scope`, `network`, and `filesystem` fields instead of stringifying nested lists.
- Added deterministic CLI diagnostics for malformed JSON, missing files, and OS read/write errors.
- Added regression tests for mixed metadata registries and malformed input handling.

## Try it

```bash
PYTHONPATH=src python3 -m mcp_registry_diff examples/registry-old.json examples/registry-new.json --format markdown --fail-on none
```
